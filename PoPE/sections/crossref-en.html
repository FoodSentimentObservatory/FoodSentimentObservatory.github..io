Activity Planc back to ToC or Class ToC

IRI: http://w3id.org/abdn/policy/pope#ActivityPlan

ActivityPlans describe the type of plan followed by an agent to guide their activity in relation to, for example, collection of data, processing of data before analysis, or analysis of the data. It is not the intent of ActivityPlan to provide a breakdown of the steps or actions that are involved in the plan; this type of information can, if desired, be represented with alternative plan representations.

Constraintc back to ToC or Class ToC

IRI: http://w3id.org/abdn/policy/pope#Constraint

A kind of role describing that the associated prov:Entity denoted some condition/state of the world further described by enties associated with role pope:Parameter. It can be further described with pope:describedAs property linking to a skos:Concept. For example, a binary tree using pope:AND_Node, pope:OR_Node, pope:XOR_Node and pope:LeafNode can be used to represent logical groupings of one or more of activity parameters.

role
has members

Leaf Nodec back to ToC or Class ToC

IRI: http://w3id.org/abdn/policy/pope#LeafNode

A concept describing a leaf node of a binary tree structure. The node is used to bind a a binary tree description of pope:Constraint to a specific pope:Parameter wich captures an aspect of the state of the world via subproperties of skos:related defined in PoPE. For eample, a parameter representing a keyword being included in a piece of text.
